# API 中转站简介

「中转站」（也叫 API 中转 / relay）是指转售大模型 API 访问能力的第三方服务：它们在上游对接 OpenAI、Anthropic、Google 等官方 API（或各种渠道的额度），对下游用户提供一个兼容官方格式的接口地址和自己发放的 API Key。

> OpenAI、Anthropic 和 Gemini API 的官方支持地区均不包含中国大陆。中转站能够建立连接，不代表使用方式符合上游服务条款，也不代表模型来源、数据处理和资金安全已经得到保证。

## 为什么会有人用中转站

- **支付方式**：部分海外平台要求其支持地区的付款方式；一些中转站接受人民币支付
- **网络接入**：部分中转站提供境内可连接的入口，但线路可能随时变化
- **模型聚合**：一个账户可能提供多家模型，不过协议、工具调用、缓存和流式事件不一定完全兼容
- **配置管理**：可以配合 [cc-switch](./tech-cc-switch.md) 保存不同接口配置；cc-switch 本身不提供模型服务

## 使用方式（通用流程）

1. 先确认运营主体、隐私政策、退款规则和上游模型来源。
2. 小额充值测试，单独创建 API Key，并设置额度或权限限制。
3. 按服务商文档配置 Base URL 和 Key。不同客户端使用的字段并不相同。
4. 用固定测试检查模型、上下文长度、工具调用和计费是否与标注一致。

## 风险与注意事项

使用中转站前应了解这些风险：

- **隐私风险**：你的全部请求内容（代码、文档、对话）都会经过中转站服务器，技术角度上看，中转站拥有完整阅读的能力，敏感内容不要发
- **资金风险**：预充值模式下，站点关停后余额通常无法追回；不要一次充太多
- **稳定性**：高峰期限速、排队、实际模型与标注不一致都可能出现
- **合规风险**：低价渠道可能来自违反上游服务条款的方式（共享账号、盗刷等），使用此类渠道的账号可能被上游封禁
- **质量差异**：同名模型在不同渠道可能有上下文截断、模型替换等情况，可用固定问题自行验证
- **密钥风险**：客户端配置、日志或同步工具可能泄露 Key；不同应用不要长期共用同一个 Key
- **数据跨境**：请求可能被发送到境外或未知子处理方，涉及个人信息、重要数据和学校内部资料时风险更高

## 使用前检查清单

- 能否核验运营主体和联系方式
- 是否公开隐私政策、日志保存期限和数据删除方式
- 是否说明上游模型来源、协议兼容范围和故障处理方式
- 是否支持小额充值、导出账单、退款或余额处理
- 是否允许关闭请求日志，是否承诺不把请求用于训练或转售
- 是否说明数据存放地区、子处理方和跨境传输情况
- 面向公众提供生成式 AI 服务时，是否具有适用的备案或登记信息

## 不推荐具体站点

本站不提供中转站推荐或充值背书。运营状态、上游渠道和服务条款变化很快，预置在某个客户端中的供应商也不代表经过审核。

学生不应通过中转站上传身份证、成绩、健康信息、未公开论文、实验数据、实习代码或学校内部系统资料。
